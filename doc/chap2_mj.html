2 Installation

Assuming you already have GAP 4.5+ installed, you can follow the steps below to install the package:

  • To get the newest version of \(\texttt{qsopt}\_\texttt{ex-interface}\), download the .zip archive from https://github.com/jayant91089/qsopt_ex-interface and unpack it using \(\texttt{unzip qsopt}\_\texttt{ex-interface-x.zip}\) in the terminal. Do this preferably inside the \(pkg\) subdirectory of your GAP 4 installation. It creates a subdirectory called \(\texttt{qsopt}\_\texttt{ex-interface}\). If you do not know the whereabouts of the \(pkg\) subdirectory, invoke the following in GAP:

GAPInfo.("RootPaths");

Look for pkg directory inside any of the paths returned.

  • Once unpacked, go to \(\texttt{qsopt}\_\texttt{ex-interface}\) directory and run the install script \(\texttt{unix-install.sh}\) from the terminal as \(\texttt{sh unix-install.sh}\). This locally installs qsopt exact and its dependencies (GMP [Gt15],libz and libbz2) in lib and include folders. Alternatively, if you have qsopt-exact and GMP already installed on your system, you can edit the Makefile inside \(\texttt{qsopt}\_\texttt{ex-interface}\) directory so that gcc finds the .so libraries. In latter case, you must manually '\texttt{make all}' from the terminal inside \(\texttt{qsopt}\_\texttt{ex-interface}\) directory.

  • Above step creates an executable \texttt{qsi} inside the \(\texttt{qsopt}\_\texttt{ex-interface}\) directory, which serves as the interface. Note that before using the package in GAP, one must edit either the environment variable \(\texttt{LD}\_\texttt{LIBRARY}\_\texttt{PATH}\) or the so that \texttt{qsi} finds the locally installed libraries.

  • One can now start using \(\texttt{qsopt}\_\texttt{ex-interface}\) by invoking

LoadPackage( "qsopt_ex-interface");

from within GAP.
